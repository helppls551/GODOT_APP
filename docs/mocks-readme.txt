#Что такое эндпоинты?
Эндпоинты это формат ответа сервера на запрос, или то, в каком формате следует посылать запрос
к серверу для выполнения какого-либо действия. Например, если хочется отправить сообщение -
для этого нужно существование эндпоинта, который может выглядеть как, например, POST /message. 
Если хочется обновить входящие сообщения, для этого нужно существование эндпоинта, который может
выглядеть как, например, GET /message. Без эндпоинтов, разработчик не знает какие данные он
получит, а какие нужно отправлять для выполнения определенных действий.
То есть, эндпоинты это единий формат общения с сервером, который должны знать разработчики.
Этот единый формат изменяется в соответсвии с требованиями разработчиков, аналитиков, дизайнеров,
и самого проекта.
В случае с нашим проектом, так как у нас нет настоящего сервера, мы имеем mock эндпоинты, т.е.
формат общения с сервером существует - просто нет самого сервера. Получается, что нет места
где следует следовать этому формату (т.к. нет сервера), но требуется обновлять этот формат
по ходу разработки.

##Этот файл представляет собой путь, краткое описание и ep-id для поиска текущих существующих 
эндпоинтов, сами json тела, примеры (форматы) ответов и запросов можно найти в /docs/mocks. 

!Эндпоинты в этом файле могут быть неактуальны или содержать ошибки. Канон это /docs/mocks.

# Mock эндпоинты (Метод/Путь/Описание/ep-id/комментарий)

GET /api/users (Получить список пользователей) (1)
POST /api/users (Создать нового пользователя) (2)

GET /api/tasks (Получить текущий список задач) (3) (status ∈ {todo, in_progress, review, done}, время unix seconds)
POST /api/tasks (Создать задачу) (4)

GET /api/time_logs (Получить логи) (5)
