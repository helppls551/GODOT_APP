#Что это такое?
api-contracts.yaml - технический документ содержащий точные форматы запросов на сервер и его ответов и являющийся единой правдой для api проекта по
типам данных, форматов запросов/ответов, нужный для снижения к минимуму неподониманий, неточностей и задержек в разработке.
mocks.json - примеры json запросов и ответов в соответсвии с контрактами

Т.к. сервером мы не располагаем, строгим следованию контрактам можно пренебречь.

#Детали mocks
Эндпоинты в этом файле ознакомительны и могут быть неточны - ровняться стоит только на сами контракты. Канон - api-contracts.yaml.
Каждый мок имеет свой уникальный ep-id (endpoint-id) и ep-description (описание), поэтому вполне можно (и стоит) ориентироваться непосредственно в нем.
Все даты в проекте хранятся как Unix timestamp в секундах.

# Эндпоинты (Метод/Путь/Описание/ep-id/комментарий)
GET /api/users (Получить список пользователей) (1)
POST /api/users (Создать нового пользователя) (2)

GET /api/tasks (Получить текущий список задач) (3) (status ∈ {todo, in_progress, review, done}, время unix seconds)
POST /api/tasks (Создать задачу) (4)

GET /api/time_logs (Получить логи) (5)
